# India-First ABDM/ABHA Planning

CareChain will focus on India first. The MVP should prepare for ABDM/ABHA-style concepts, but we will not claim official ABDM integration until real API access and compliance review are completed.

## Positioning

Use:

- India-first healthcare platform
- ABHA-style health identity ready
- Consent-first medical record sharing
- Designed for future ABDM alignment

Avoid:

- ABDM certified
- Official ABHA integrated
- Government approved
- Official health locker

## Patient Identity Fields

- Full name
- Date of birth
- Gender
- Mobile number
- Optional ABHA number placeholder
- Optional ABHA address placeholder
- Wallet address
- CareChain patient ID

## Doctor Identity Fields

- Doctor name
- Specialty
- Registration number placeholder
- Clinic or hospital
- Wallet address
- Verification status

## Facility Fields

- Facility name
- Facility type
- City
- State
- Admin user
- Verification status

## Consent Model

CareChain should track:

- Access request created
- Patient approved
- Patient rejected
- Access granted
- Access revoked
- Access expired
- Emergency access granted
- Emergency access expired

## Medical Record Metadata

Each medical record should eventually include:

- Record type
- File name
- Patient owner
- Uploaded by
- Upload date
- Storage CID
- File hash
- Blockchain transaction hash
- Verification status

## MVP Rule

For v2.0, we prepare the structure for ABDM/ABHA-style workflows, but we do not build official ABDM integration yet.
