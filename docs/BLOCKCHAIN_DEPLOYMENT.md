# Blockchain Deployment Notes

CareChain uses Solidity smart contracts for record integrity and access-related blockchain workflows.

## Current Contract Scope

The `RecordRegistry` contract stores record verification metadata:

- IPFS CID
- Record type
- SHA-256 file hash
- Timestamp
- Patient wallet address through `msg.sender`

Medical files are never stored on-chain.

## Record Integrity Flow

```txt
Medical file
→ SHA-256 file hash generated by backend
→ File encrypted by backend
→ Encrypted file uploaded to IPFS
→ IPFS CID + record type + file hash submitted to RecordRegistry
→ MongoDB stores record metadata and transaction hash
